## 新增需求

### 需求：统一的创建/编辑表单

物品表单页应通过基于路由的模式检测同时服务创建（`/items/create`）和编辑（`/items/:id/edit`）。在编辑模式下，表单应预填充现有物品数据。

页面标题和返回导航由全局 `MainLayout` 根据路由 `meta.title` 和 `meta.showBack` 统一渲染，表单页自身不再包含独立标题栏和返回按钮。

#### 场景：用户创建新物品

- **WHEN** 用户导航到 `/items/create`
- **THEN** 显示空表单，并预填充今天日期（R010）

#### 场景：用户编辑现有物品

- **WHEN** 用户导航到 `/items/:id/edit`
- **THEN** 表单填充物品的当前数据

### 需求：名称校验（R001）

物品名称字段应为必填，并限制在 2–50 个字符。在失焦或提交时，无效输入应显示行内错误信息。

#### 场景：用户提交无效名称

- **WHEN** 用户留空名称或输入 "A"（1 个字符）
- **THEN** 表单显示 "物品名称需 2-50 个字符"
- **WHEN** 用户输入 51 个字符
- **THEN** 表单显示相同错误并阻止提交

### 需求：分类选择器

表单应提供一个从 categories store 填充的分类下拉选择器。未选择分类时显示占位符 "请选择分类"。

#### 场景：用户选择分类

- **WHEN** 用户打开分类下拉框
- **THEN** 列出所有现有分类
- **WHEN** 用户选择一个分类
- **THEN** 所选分类绑定到表单

### 需求：照片上传（R006/R007）

表单应通过 `el-upload` 允许上传最多 3 张照片。每张照片应压缩至宽度 ≤1280px，质量 80%，目标 ≤200KB。照片应显示为带移除按钮的缩略图（100×100px，占比较大）。点击缩略图打开预览弹窗。

#### 场景：用户上传照片

- **WHEN** 用户选择一张照片
- **THEN** 照片被压缩并作为 base64 缩略图添加
- **WHEN** 已存在 3 张照片
- **THEN** 上传按钮被禁用

#### 场景：用户移除照片

- **WHEN** 用户点击照片上的移除图标
- **THEN** 照片从表单中移除

### 需求：存放日期选择器

表单应提供一个存放日期的日期选择器。如果用户未选择日期，表单初始化时应自动填充今天日期（R010）。

#### 场景：用户看到默认日期

- **WHEN** 用户打开创建表单
- **THEN** 存放日期字段显示今天日期

### 需求：储物柜选择器

表单应提供一个从 cabinets store 填充的下拉选择器。所选储物柜的 ID 应与物品一起存储。

#### 场景：用户变更储物柜

- **WHEN** 用户在编辑模式下选择不同的储物柜
- **THEN** 保存时，旧和新储物柜的计数都会更新（R011）

### 需求：备注文本域

表单应提供一个多行文本域用于可选备注，最多 200 个字符。

#### 场景：用户添加备注

- **WHEN** 用户在备注字段输入 "去年双十一购入"
- **THEN** 文本与物品一起保存

### 需求：保存与导航

表单应提供一个主操作 "保存" 按钮。保存成功后，显示成功 toast 并导航回 `/items`。校验失败时，第一个无效字段应获得焦点。

#### 场景：用户保存新物品

- **WHEN** 用户填写有效数据并点击保存
- **THEN** 物品被创建，储物柜计数递增，用户带着成功 toast 返回列表

### 需求：未保存更改守卫

如果用户尝试通过浏览器返回、系统返回按钮或其他方式离开表单路由（未点击保存），应出现确认弹窗："有未保存的更改，确定要离开吗？"

点击保存按钮并成功提交后，守卫应自动跳过，不再弹窗，直接返回 `/items`。

守卫通过 `onBeforeRouteLeave`（从 `vue-router` 导入）配合一个保存成功标志位实现：保存成功后置标志位为 `true`，守卫检测到标志位直接放行。

#### 场景：用户带着未保存更改返回

- **WHEN** 用户修改了表单并点击系统返回按钮触发路由离开
- **THEN** 导航前出现确认弹窗

#### 场景：用户保存成功后离开

- **WHEN** 用户点击保存并提交成功
- **THEN** 成功 toast 出现，直接返回 `/items`，不弹出确认对话框
