## 新增需求

### 需求：物品列表页布局

在 `/items` 的物品列表页应显示一个可垂直滚动的物品卡片列表，顶部有一个固定的搜索/筛选栏。页面应使用 Element Plus 组件和 Tailwind CSS 进行布局。

#### 场景：用户打开物品列表

- **WHEN** 已认证用户导航到 `/items`
- **THEN** 页面显示搜索输入框、分类筛选下拉框、日期范围选择器，以及按 `createdAt` 降序排列的物品卡片列表

### 需求：物品卡片展示

每个物品卡片应显示：主照片缩略图（或占位图）、加粗标题的物品名称、分类标签、储物柜位置、存放日期。卡片应可点击以导航到物品详情页。

#### 场景：用户查看物品列表

- **WHEN** store 中存在物品
- **THEN** 每个卡片显示 `需求文档.md` §4.2 中定义的字段

### 需求：关键字搜索

物品列表应支持实时关键字筛选，具备 300ms 防抖。关键字应通过部分包含匹配物品名称。

#### 场景：用户搜索物品

- **WHEN** 用户在搜索输入框中输入 "螺丝刀"
- **THEN** 仅显示名称中包含 "螺丝刀" 的物品

### 需求：分类筛选

物品列表应提供一个下拉选择器，按单个分类筛选。提供"全部"选项以重置筛选。

#### 场景：用户按分类筛选

- **WHEN** 用户从分类下拉框中选择 "工具"
- **THEN** 仅显示 `categoryId` 匹配 "工具" 的物品

### 需求：时间范围筛选

物品列表应提供一个日期范围选择器，按存放日期筛选（起始和结束均包含）。清空选择器应显示所有物品。

选择器使用 `el-date-picker` 配合 `type="daterange"` 和 `value-format="YYYY-MM-DD"`，绑定值为 `[string, string]` 字符串数组，直接作为 `dateStart` / `dateEnd` 传给 `ItemFilters`，无需额外转换。

#### 场景：用户按日期范围筛选

- **WHEN** 用户在日期选择器中选择 2025-01-01 至 2025-01-31
- **THEN** 仅显示 `storageDate` 在包含范围内（含边界）的物品

### 需求：组合筛选逻辑

当多个筛选同时激活时，物品列表应应用 AND 逻辑：物品必须满足所有活跃筛选条件才能被显示。

#### 场景：用户应用多个筛选

- **WHEN** 用户搜索 "螺丝刀" 并同时选择分类 "工具"
- **THEN** 仅显示同时满足两个条件的物品

### 需求：滑动删除手势

物品卡片应支持左滑手势以露出删除按钮。点击删除按钮应触发确认弹窗（R012）。非触摸设备提供悬停删除按钮作为备用方案。

#### 场景：用户滑动删除物品

- **WHEN** 用户向左滑动物品卡片并点击删除
- **THEN** `ElMessageBox.confirm` 出现；确认后，物品被删除且储物柜计数递减

### 需求：加载更多分页

物品列表初始应显示 20 个物品，并提供一个"加载更多"按钮。支持触底自动加载。当所有物品加载完毕后，应出现"没有更多"提示。

#### 场景：用户加载更多物品

- **WHEN** 用户点击"加载更多"
- **THEN** 接下来的 20 个物品被追加到列表
- **WHEN** 用户滚动快到底部时
- **THEN** 接下来的 20 个物品自动被追加到列表
