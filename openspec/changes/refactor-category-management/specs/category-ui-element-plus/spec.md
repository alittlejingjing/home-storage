## 新增需求

### 需求：分类页面仅使用 Element Plus 组件
系统应仅使用 Element Plus 组件渲染交互元素，Tailwind CSS 负责布局，并使用 `@element-plus/icons-vue` 作为图标库。

#### 场景：分类列表展示
- **当** 用户导航到 `/categories` 页面时
- **则** 分类列表使用 `el-card` 容器渲染，外层使用 Tailwind CSS 间距
- **并** 每张卡片展示：颜色标签圆点、分类名称、物品数量、操作按钮（上移/下移、编辑、删除）

#### 场景：空状态
- **当** 不存在任何分类时
- **则** 渲染空状态，使用 `el-empty` 或 Tailwind 布局的自定义空状态块，包含 `CollectionTag` 图标和描述文字

#### 场景：行内编辑
- **当** 用户点击分类的编辑按钮时
- **则** 分类行切换为编辑模式，显示 `el-input` 输入框和确认/取消按钮（使用 `Check` 和 `Close` 图标）
- **并** 按下 Enter 键确认编辑

#### 场景：新增分类
- **当** 用户在底部固定输入区输入名称并点击添加或按下 Enter 时
- **则** 通过 Store/Repository 创建新分类
- **并** 输入框被清空

### 需求：删除确认及策略选择
系统应在删除分类时展示确认弹窗，并允许用户选择如何处理关联物品。

#### 场景：删除无关联物品的分类
- **当** 用户点击删除一个物品数量为 0 的分类时
- **则** `ElMessageBox.confirm` 弹窗提示"确认删除此分类？删除后不可恢复"
- **并** 确认后删除该分类

#### 场景：删除有关联物品的分类
- **当** 用户点击删除一个物品数量大于 0 的分类时
- **则** `ElMessageBox.confirm` 弹窗显示："该分类下还有 N 件物品，请选择处理方式"
- **并** 弹窗中包含 `el-radio-group`，提供两个选项："一并删除"（`DELETE_ITEMS`）和 "转移至未分类"（`MOVE_TO_UNCATEGORIZED`）
- **并** 用户必须选择一项后方可确认
- **并** 确认后应用所选策略

### 需求：通过 Element Plus 实现消息通知
系统应使用 `ElMessage` 替代旧版 `useToast` 处理所有成功/错误通知。

#### 场景：添加成功
- **当** 分类成功添加时
- **则** 展示 `ElMessage.success('分类添加成功')`

#### 场景：添加失败（名称重复）
- **当** 添加分类因名称重复而失败时
- **则** 展示 `ElMessage.error('分类名称已存在')`

#### 场景：校验错误
- **当** 分类名称为空或超过 20 个字符时
- **则** 展示 `ElMessage.warning('分类名称不能为空且不超过20个字符')`
