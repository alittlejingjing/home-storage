## 新增需求

### 需求：搜索页面使用 Element Plus 组件替代 Reka UI
系统应仅使用 Element Plus 组件渲染搜索页面的交互元素，Tailwind CSS 负责布局，并使用 `@element-plus/icons-vue` 作为图标库。

#### 场景：搜索输入
- **当** 用户进入搜索页面时
- **则** 页面顶部展示 `el-input` 输入框，带 `Search` 前缀图标和 `clearable` 清除按钮
- **并** 输入框 placeholder 为"搜索物品、储物柜..."
- **并** 输入防抖 300ms 后触发搜索

#### 场景：Tab 切换（物品/储物柜）
- **当** 用户点击"物品"或"储物柜"标签时
- **则** 使用 `el-tabs` 组件展示两个 Tab 面板
- **并** 当前选中的 Tab 显示墨绿色下划线
- **并** Tab 切换时内容区域平滑过渡

#### 场景：搜索结果列表（物品）
- **当** 搜索结果包含物品时
- **则** 每个物品以 `el-card` 展示，内含物品图片（或默认 `Box` 图标）、名称、分类标签、所属储物柜
- **并** 分类标签使用与分类管理页一致的颜色方案（温暖有机纯色）
- **并** 点击卡片跳转物品详情页

#### 场景：搜索结果列表（储物柜）
- **当** 搜索结果包含储物柜时
- **则** 每个储物柜以 `el-card` 展示，内含 `OfficeBuilding` 图标、名称、位置描述
- **并** 点击卡片跳转储物柜详情页

#### 场景：空状态
- **当** 搜索关键词无结果时
- **则** 展示自定义空状态：对应图标（`Search` 或 `Box`/`OfficeBuilding`）+ 提示文字
- **当** 页面无关键词且数据为空时
- **则** 展示引导空状态"快去添加第一件物品/储物柜吧"

### 需求：搜索页面样式与首页统一
系统应确保搜索页面的背景、卡片、Tab 等视觉风格与首页 `fd-home` 保持一致。

#### 场景：页面背景
- **当** 用户进入搜索页面时
- **则** 页面背景为 `#f3f9f6`，带蓝绿色径向渐变光晕

#### 场景：卡片样式
- **当** 搜索结果展示时
- **则** 每个结果卡片使用毛玻璃效果（半透明白底 + backdrop-filter）
- **并** 卡片圆角为 1rem
- **并** 卡片带 3px 顶部装饰色带（蓝绿渐变）
- **并** 卡片悬浮时上移 2px + 阴影增强

#### 场景：Tab 选中样式
- **当** Tab 被选中时
- **则** 活动指示器为墨绿色 `#2d8a78`
- **并** 选中文本加粗

### 需求：彻底移除 Reka UI 和 lucide-vue-next 依赖
系统中 Search.vue 页面不再引用 `@/components/ui` 和 `lucide-vue-next`。

#### 场景：代码检查
- **当** 在 Search.vue 中搜索 `@/components/ui` 时
- **则** 无任何匹配结果
- **当** 在 Search.vue 中搜索 `lucide-vue-next` 时
- **则** 无任何匹配结果
