## 3. 配置与工厂

### 3.1 数据源配置

**文件**：`frontend/src/config/dataSource.ts`

```typescript
export type DataSourceMode = 'local' | 'remote'

export const DATA_SOURCE: DataSourceMode =
  (import.meta.env.VITE_DATA_SOURCE as DataSourceMode) || 'local'

export const API_BASE_URL = import.meta.env.VITE_API_BASE_URL || '/api'
```

| 环境变量 | V1 | V2 |
|----------|----|----|
| `VITE_DATA_SOURCE` | `local` | `remote` |
| `VITE_API_BASE_URL` | 可不配 | 后端网关地址 |

### 3.2 Repository 工厂（全模块）

**文件**：`frontend/src/repositories/index.ts`

| 工厂方法 | V1 返回 | V2 返回 |
|----------|---------|---------|
| `getItemRepository()` | `localItemRepository` | `httpItemRepository` |
| `getCabinetRepository()` | `localCabinetRepository` | `httpCabinetRepository` |
| `getCategoryRepository()` | `localCategoryRepository` | `httpCategoryRepository` |
| `getDashboardRepository()` | `localDashboardRepository` | `httpDashboardRepository` |
| `getSearchRepository()` | `localSearchRepository` | `httpSearchRepository` |
| `getAuthRepository()` | `localAuthRepository` | `httpAuthRepository` |
| `getBackupRepository()` | `localBackupRepository` | `httpBackupRepository` |
| `getPhotoRepository()` | `indexedDbPhotoRepository` | `httpPhotoRepository`（或仍走 item/cabinet 照片 API） |

```typescript
export function getItemRepository(): IItemRepository {
  return DATA_SOURCE === 'remote' ? httpItemRepository : localItemRepository
}
```

## 6. V2 HTTP 层约定

### 6.1 统一响应

```typescript
export interface Result<T> {
  code: number
  message: string
  data: T | null
}
```

后端 `com.thunisoft.homestorage.common.Result` 字段一致。

### 6.2 HttpRepository 实现模板

```typescript
async list(query: ItemQuery): Promise<PageResult<Item>> {
  const { data } = await request.get<Result<PageResult<Item>>>('/api/items', { params: query })
  if (data.code !== 200) throw new Error(data.message)
  return data.data!
}
```

各模块 **§5 API 表** 已列出「Repository 方法 ↔ HTTP」完整映射，实现时逐方法照抄路径。

### 6.3 后端工程（V2）

```
backend/src/main/java/com/thunisoft/homestorage/
├── controller/    # 与 api/*.ts 路径一致
├── service/       # 与详细设计 §4.1 一致
├── mapper/        # MyBatis Plus BaseMapper
├── entity/        # @TableName 与 SQL 一致
├── dto/ / vo/
└── common/        # Result、异常码
```

## 7. Pinia Store 约定

```typescript
const repo = getItemRepository()

async function loadList(query: ItemQuery) {
  const page = await repo.list(query)
  items.value = page.records
}
```

切换 V2 **无需改 Store 方法签名**，仅环境变量变更。
